DNSSEC и что делать, если DNSSEC не поддерживается (пошагово)

Проблема
-------
На хостинге, где размещается DNS для вашего домена (например, если вы используете Netlify DNS), может не быть поддержки DNSSEC. Без DNSSEC ответы DNS могут быть перехвачены или подменены между рекурсивным резолвером и авторитативным сервером — это риск для целостности путей к сертификатам/сайтами.

Цель
-----
Снизить риск атак, связанных с отсутствием DNSSEC, и ограничить возможность выдачи поддельных сертификатов.

Рекомендованные шаги
---------------------
1) Проверить текущее состояние DNS
   - Проверьте, где расположен DNS (имена серверов). Если у вас Netlify DNS — у Netlify нет поддержки DNSSEC для некоторых вариантов. Лучше иметь DNS у провайдера, поддерживающего DNSSEC.

2) Перейти на DNS-провайдера с поддержкой DNSSEC
   - Популярные варианты: Cloudflare (DNSSEC поддерживается), Amazon Route53 (частично, требует настройки у регистратора), NS1, Gandi и пр.
   - Вариант: перенести делегирование домена на Cloudflare — Cloudflare поддерживает DNSSEC и выдаёт DS-запись, которую нужно добавить у регистратора.

3) Включить DNSSEC у DNS-провайдера и добавить DS-запись у регистратора
   - Включите DNSSEC в панели DNS-провайдера. Скопируйте DS (Digest, Algorithm, Digest Type, Key Tag).
   - На стороне регистратора добавьте DS-запись (Domain Signer) в настройки домена.
   - Проверьте через 24–48 часов с помощью: dig +dnssec example.com or online  https://www.dnssec-analyzer.verisign.com/

4) Ограничьте выдачу сертификатов с помощью CAA
   - Добавьте CAA-записи, чтобы разрешить выпуск сертификатов только доверенным CA.
   - Пример (разрешить только Let's Encrypt):
     - Type: CAA, Name: @, Value: 0 issue "letsencrypt.org"

5) Включите заголовки защиты на стороне сайта (HSTS, CSP, X-Frame-Options и пр.)
   - Для Netlify можно создать файл `_headers` в корне сайта (я добавил примерный в репозиторий). Важные заголовки:
     - Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
     - Content-Security-Policy: (ваша политика)
     - X-Frame-Options: DENY
     - X-Content-Type-Options: nosniff
     - Referrer-Policy: no-referrer
     - Permissions-Policy: camera=(), microphone=(), geolocation=()
   - Эти заголовки не заменяют DNSSEC, но снижают атаку на последствия подмены DNS.

6) Настроить мониторинг и проверки
   - Включите уведомления о новых сертификатах (Certificate Transparency) и используйте сервисы мониторинга, чтобы отслеживать неожиданную выдачу сертификатов.
   - Периодически проверяйте DNSSEC через dig/online-инструменты.

7) Если регистратор не поддерживает добавление DS-записей
   - Перенесите домен к регистратору, который поддерживает DS-записи, или используйте DNS-провайдера, который может делегировать через делегирование NS (зависит от политик регистратора).

Проверка (коротко)
------------------
- После включения DNSSEC:
  - `dig +dnssec yourdomain.com` — убедиться, что присутствует RRSIG/AD флаг
  - Онлайн: https://dnssec-debugger.verisign.com/ или https://www.dnsviz.net/

Примечания
----------
- DNSSEC защищает целостность DNS-ответов, но не шифрует их.
- Если вы используете Netlify только как хост статики, вы можете хранить DNS у внешнего провайдера (с поддержкой DNSSEC) и указывать в Netlify кастомный домен.

Если хотите, я могу:
- Проверить, где сейчас размещён DNS для вашего домена (укажите домен) и дать пошаговую инструкцию конкретно для вашего регистратора/провайдера.
- Подготовить готовые CAA- и DS-примеры для вашего домена.
