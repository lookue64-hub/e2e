# 🔐 E2E Telega - Анонимный шифровальщик

Защищённый web-app для локального шифрования сообщений с использованием современной криптографии (AES-GCM + HMAC + PBKDF2).

## 📁 Структура файлов

### Frontend (Web-приложение)
- **index.html** — главная HTML-страница с интерфейсом
- **app.js** — JavaScript логика шифрования/расшифровки и управление UI
- **_headers** — заголовки безопасности для Netlify/GitHub Pages

### Backend (Telegram Bot)
- **main.py** — bot.py для Telegram с интеграцией WebApp

### Документация
- **SECURITY_DNSSEC.md** — инструкции по DNSSEC и DNS-безопасности
- **README.md** — этот файл

## 🔐 Особенности безопасности

### Криптография
- **Шифрование:** AES-256-GCM (аутентифицированное шифрование)
- **Вывод ключа:** PBKDF2 с 400,000 итерациями (SHA-256)
- **Аутентификация:** HMAC-SHA256
- **Соль:** 16 байт (случайные для каждого шифрования)
- **IV (Nonce):** 12 байт (случайный для каждого шифрования)

### Веб-безопасность

#### Content Security Policy (CSP)
- Блокирует встроенные скрипты (`script-src 'self'`)
- Запрещает фреймирование (`frame-ancestors 'none'`)
- Разрешает только локальные ресурсы

#### Заголовки защиты
- **X-Frame-Options:** DENY (защита от clickjacking)
- **X-Content-Type-Options:** nosniff (защита от MIME-перехвата)
- **Referrer-Policy:** no-referrer (скрытие рефереров)
- **Cache-Control:** no-store, no-cache (запрет кэширования)
- **Strict-Transport-Security:** (HSTS для HTTPS)

### Анонимность
- Нет отправки данных на сервер
- Нет интеграции с Telegram WebApp API (отключена)
- Нет логирования в браузере
- Все вычисления локальные

## 📊 Ограничения и планы

### Текущие ограничения
- Шифрование только локально (нет серверной части)
- Поддержка только текстовых сообщений
- Зависимость от HTTPS для полной безопасности

### Возможные улучшения
- Поддержка файлов
- Синхронизация между устройствами (зашифрованная)
- Поддержка QR-кодов для обмена ключами

## 🛡️ DNS/DNSSEC безопасность

Сейчас сайт может быть размещен на Netlify/GitHub Pages, которые не поддерживают DNSSEC. Для полной безопасности:

1. Переместите DNS на Cloudflare (поддерживает DNSSEC)
2. Добавьте DS-запись у регистратора
3. Добавьте CAA-записи для контроля выпуска сертификатов

Подробнее см. `SECURITY_DNSSEC.md`
